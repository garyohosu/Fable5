# 技術調査の使用例

## テンプレート

```
Opus 4.8 Fable-like Modeで技術調査してください。

調べたいこと：
{調査テーマ}

前提：
{環境、バージョン、制約など}

欲しい結果：
- 結論
- 理由
- 使える場合
- 使えない場合
- 注意点
- 実行手順
- 参考にすべき公式情報

条件：
- 不明点は不明と書く
- 推測と事実を分ける
- 日本語で説明する
```

## 記入例：電子工作

```
Opus 4.8 Fable-like Modeで技術調査してください。

調べたいこと：
ESP32でBME280（温湿度気圧センサー）をI2C接続する方法と注意点

前提：
- Arduino IDE 2.x を使用
- ESP32 DevKitC
- 電源は3.3V

欲しい結果：
- 結論
- 配線（ピン対応）
- 使うライブラリと実行手順
- ハマりやすい点（プルアップ、アドレス違いなど）
- 参考にすべき公式情報

条件：
- 不明点は不明と書く
- 推測と事実を分ける
- 日本語で説明する
```

## 記入例：ツール運用相談

```
Opus 4.8 Fable-like Modeで技術調査してください。

調べたいこと：
Claude Code、Codex、Gemini CLIを併用する場合の使い分け方針

前提：
- 個人開発
- 月額コストは抑えたい
- 主な作業はPythonスクリプトとNote記事作成

欲しい結果：
- 結論（どの作業をどのツールに割り当てるか）
- 理由
- 注意点

条件：
- 推測と事実を分ける
- 日本語で説明する
```

## 期待される動き

1. 調査テーマを一文で再定義し、前提を整理する
2. 事実（ドキュメント・仕様に基づく）と推測（経験則）を明確に分ける
3. バージョン依存や環境差分など、結論が変わりうる条件を明示する
4. 不明な点は「不明」と書き、確認方法を添える
