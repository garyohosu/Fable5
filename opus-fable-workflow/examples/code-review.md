# コードレビューの使用例

## テンプレート

```
Opus 4.8 Fable-like Modeでコードレビューしてください。

対象：
{ファイル名またはコード}

確認してほしい観点：
- バグ
- 仕様漏れ
- 例外処理
- 境界条件
- セキュリティ
- 保守性
- テスト不足
- 最小修正案

条件：
- 重大な問題から順に指摘する
- すぐ直せる形で提案する
- 不要なリファクタリングは避ける
- 日本語で説明する
```

## 記入例：複数ファイルのレビュー

```
Opus 4.8 Fable-like Modeでコードレビューしてください。

対象：
src/sensor_reader.py（シリアルポートから測定値を読む）
src/logger.py（測定値をCSVに追記する）
仕様：1秒ごとに測定し、通信エラー時は3回リトライして記録を続ける

確認してほしい観点：
- バグ
- 仕様漏れ（リトライ回数、エラー時の継続）
- 例外処理（ポート切断、CSV書き込み失敗）
- 境界条件（空データ、不正な値）
- 最小修正案

条件：
- 重大な問題から順に指摘する
- すぐ直せる形で提案する
- 不要なリファクタリングは避ける
- 日本語で説明する
```

## 期待される出力形式

1. **重大**：データ欠損・クラッシュにつながる問題（修正案付き）
2. **中**：仕様不一致、例外処理漏れ
3. **軽微**：命名、保守性
4. テスト不足の指摘と、最低限追加すべきテストケース

## 注意

- 指摘は重大度順。全部を同じ重みで並べない
- 修正案は差分中心。ファイル全文の書き直しは求められない限り出さない
- 「動いているコードの好みによる書き換え」は提案されない
