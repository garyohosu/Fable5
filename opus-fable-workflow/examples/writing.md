# 文章・記事作成の使用例

## テンプレート

```
Opus 4.8 Fable-like Modeで文章を作成してください。

目的：
{何のための文章か}

読者：
{誰に向けた文章か}

使う場所：
{Note、Quora、メール、社内文書、SNSなど}

条件：
- 結論を先に書く
- 誤解されそうな点を避ける
- 必要なら言い回しを柔らかくする
- 論理の飛躍をなくす
- 最後に読みやすさを確認する
- 日本語で出力する

出力形式：
{全文、見出し付き、短文、メール形式など}
```

## 記入例：Note記事

```
Opus 4.8 Fable-like Modeで文章を作成してください。

目的：
Claude CodeとCodexの使い分けを初心者向けに解説するNote記事の下書き

読者：
AIコーディングツールを使い始めたばかりの個人開発者

使う場所：
Note

条件：
- 結論を先に書く
- 誤解されそうな点を避ける
- 論理の飛躍をなくす
- 最後に読みやすさを確認する
- 日本語で出力する

出力形式：
タイトル案3つ + 見出し付き全文（2000字程度）
```

## 記入例：炎上リスクチェック

```
Opus 4.8 Fable-like Modeで文章を作成してください。

目的：
以下のSNS投稿文の炎上リスクと論理の穴をチェックし、修正案を出す

読者：
技術系のフォロワー

使う場所：
X（Twitter）

条件：
- 誤解されそうな点を具体的に指摘する
- 断定しすぎている箇所を洗い出す
- 修正版はそのまま貼れる形にする
- 日本語で出力する

投稿文：
{ここに投稿文を貼る}
```

## 期待される動き

1. 読者像と目的を一文で再定義する
2. 構成（導入 → 主張 → 具体例 → まとめ）を決めてから書く
3. 誤解されそうな表現・炎上リスクを確認してから最終版を出す
4. 記事の場合は「下書き → 改善 → 最終版」を分けて、一度に全部を長く出さない
