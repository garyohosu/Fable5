# Claude Code作業の使用例

## テンプレート

```
Opus 4.8 Fable-like Modeで作業してください。

目的：
{ここに目的を書く}

対象：
{対象ファイル、文章、コード、資料などを書く}

制約：
- Fable 5は使わない
- Opus 4.8で可能な範囲で高品質に処理する
- 作業を分解する
- 不明点は仮定を明示して進める
- 最後に自己レビューする
- 出力は日本語

やってほしいこと：
1. 目的を整理する
2. 問題点やリスクを洗い出す
3. 作業方針を決める
4. 実際に修正または作成する
5. 結果を確認する
6. 最終報告を出す
```

## 記入例：バグ修正

```
Opus 4.8 Fable-like Modeで作業してください。

目的：
CSVアップロード時に日本語ファイル名で文字化けするバグを直したい

対象：
app/upload.py と app/utils/filename.py

制約：
- Fable 5は使わない
- Opus 4.8で可能な範囲で高品質に処理する
- 作業を分解する
- 不明点は仮定を明示して進める
- 最後に自己レビューする
- 出力は日本語

やってほしいこと：
1. 目的を整理する
2. 問題点やリスクを洗い出す
3. 作業方針を決める
4. 実際に修正または作成する
5. 結果を確認する
6. 最終報告を出す
```

## 期待される動き

1. 対象ファイルと呼び出し元を読み、影響範囲を確認する
2. 原因候補（エンコーディング指定漏れ、OS差分など）を優先順位付きで整理する
3. 最小変更で修正し、確認手順（再現テストのやり方）を示す
4. コミット・pushはユーザーの指示があるまで行わない
5. 変更内容を簡潔に報告する

## 注意

- 電子工作や製造業向けのスクリプト（シリアル通信、CSV集計、装置ログ解析など）でも同じ形式が使える
- 「ついでにリファクタリング」は依頼しない限り行われない
